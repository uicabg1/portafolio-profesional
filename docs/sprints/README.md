# Sprints activos

Esta carpeta concentra la planificación operativa vigente del proyecto.

## Propósito

- documentar el sprint actual o el siguiente sprint ejecutable
- separar trabajo activo de los runbooks históricos
- enlazar backlog, arquitectura y operaciones sin duplicar estado

## Regla de uso

- `docs/sprints/` describe trabajo vivo o inmediatamente ejecutable
- `docs/archive/sprints/` conserva la historia de cómo se construyó el proyecto
- cuando un sprint deja de ser referencia operativa, puede archivarse en `docs/archive/sprints/`
